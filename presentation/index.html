Tech Stack at a Glance

Current repo snapshot: Next.js 16, FastAPI, Postgres, Chroma, Docker Compose, and runlocal.sh.

Project Structure high-level system map 
Platform/
├── frontend/
│   ├── app/               # feed, search, sources, saved, wiki
│   ├── components/        # Shared UI building blocks
│   ├── hooks/             # Client state helpers
│   └── lib/               # API client and sync utilities
├── backend/
│   ├── app/
│   │   ├── main.py        # API startup and lifecycle
│   │   ├── api/routes/    # news, search, research, wiki
│   │   ├── services/      # RSS, extraction, clustering, agents
│   │   └── data/          # curated RSS source catalog
│   └── rss_parser_rust/   # parser acceleration path
├── docker-compose.yml     # postgres, chromadb, embedding, api, frontend
└── runlocal.sh            # local Postgres, Chroma, embedding, gunicorn, Next
Layer Technology
Frontend Next.js 16, React 19, TypeScript, Tailwind CSS, shadcn/ui, React Query
Backend FastAPI, Python, SQLAlchemy async, LangChain, gunicorn or uvicorn
Storage PostgreSQL 17, ChromaDB, curated RSS catalog
AI OpenRouter or Gemini, embedding service, verification flows
Infra Docker Compose and runlocal.sh for local development
01

Client

Feed, search, sources, saved items, and wiki views run in the App Router.

02

API

FastAPI routes cover news, trending, research, verification, search, and wiki surfaces.

03

Logic

Services handle RSS ingestion, article extraction, clustering, scoring, and research agents.

04

Data

Postgres stores product state while Chroma supports embeddings, recall, and topic grouping.

05

Runtime

Docker Compose runs five services; runlocal.sh can start Postgres, Chroma, embedding, backend, and frontend directly.

Key Readout

Next.js 16 frontend, FastAPI services, Postgres plus Chroma, Docker and local script orchestration.

The current repo is organized around clear boundaries between UI, route handlers, service logic, storage, and AI-assisted research workflows.
